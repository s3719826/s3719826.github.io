Project Idea

Delivery Tracker

This project idea would be the creation of a 3rd party delivery tracker for any orders that you order online to be delivered to your place on websites such as Amazon and Kogan. This would allow for a webpage under a user’s account to view tracking from different companies and locations in a simpler way rather than having to connect to the different websites. This would require integration of multiple websites as the website would require the delivery information to be able to display it. The motivation behind this project is due to the increase in the need in ordering products from store to our doors due to the trying times that the corona virus has created, with the increase in use of websites such as Amazon and many companies doing deliveries there are now too many websites required to look at to view your deliveries at as depending on which website you order at will have its own delivery tracker, well the idea is to combine them all into one comfortable viewer that allows you to see the status of your orders no matter which company it is which allow the users of multiple companies convenience. This product would allow users to sign up with users to view their deliveries. On the main page you would have a list of your websites which you are ordering from that are clickable and direct you to a page where you can view just that companies’ deliveries, as well as having a total amount of current deliveries pending on the main page. Another addition to the main page would be a list of overdue deliveries that you may need to follow up from the company that is delayed, the company’s name and order ID would be displayed and therefore would allow for the user to search for the order at ease, our website will also supply a link that directly goes to the website. You would also be able to click on a dropdown which would allow you to view all deliveries from different companies ordered by companies or ordered by delivery date. There would also be the ability to look at your profile and edit it as required. One of the main features of this website is the button that redirects you to the listed delivery companies for the user. Delivery TrackerTM would have an API to receive information about the order and the website which is supplying the delivery would require to send updates to the API to update the delivery status and therefore the project would require to work together with companies to make it successful. There will be a basic API endpoint, but there would also be API endpoints specialised to different companies’ needs, making it less generic. At a certain point in the future we would like this to become a mobile app that sends updates as orders get updated to each customer it would also allow for users to view it on their phone in a phone friendly application. Something else that we would like to implement is the ability to control the orders from our website, operations such as cancelling, updating delivery location at the earlier stages but this would require the websites permission as well as an endpoint from their end. The last thing we would like to implement which would increase the reasons in which the user would like to use Delivery TrackerTM is to create a live map for all deliveries, which would show you either the location that they are at with the ability to click to view the information on the map, so without it moving to another page, or if its on the move to see the pathway at which it is going to, this can be a direct line as it wouldn’t be exact location, which would show the departure date as well as an estimated time to arrival. The Tracker would be free for users to use but would cost companies as per contract and size of company and will have support for any special occurrences of issues. The tools required for Delivery TrackerTM would be as simple as javascript, html,css for the languages as well as Node.js as a way to run it on a server, we would also require a testing framework such as JEST to ensure that the code is running correctly with no issues, it will also require a database such as SQL to keep all the user information as well as order information. It would also require very heavy protection so the information doesn’t leak as this will have private information such as delivery addresses which would cause major privacy issues for companies. Hardware required would be servers that would be most likely hosted on AWS which would be covered by ads displayed on the website, although initially as the clientele would be small something like Heroku or the base AWS will work. For writing this project I would require skills in Javascript, html, css, Node.js, JEST as well as SQL for the database most of these languages / tools have easy to use documentation that can be found on the internet and is feasible to find and learn. AWS which may be used as server might require a bit more research as there are many aspects to it and finding the right type of instance for the server will be difficult depending on the progress of the application, as there may be endpoints that would require different types of AWS services. If this project were to succeed it would allow the user to view any order of any company integrated with DeliveryTrackerTM and allow for live tracking as well as updating orders and allow for a more comfortable viewing with phone updates as status’ changes. This would make the use of many different websites and logins simpler. This implementation could become the biggest way to track deliveries and would be a great way for companies that do not have the ability to develop their own tracker have the ability to use a tracker.
